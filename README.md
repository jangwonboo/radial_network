# Radial Network Diagram — Strategy House

Interactive radial network diagram of 39 strategic initiatives on three concentric rings by time
horizon (short = inner, mid = middle, long = outer), connected by cross-initiative dependency
links. Single self-contained HTML file (D3 via CDN) — no server, no build step, no database.

**The whole folder is the deployable site. `index.html` is the single source of truth.**

## Structure
```
radial-network/
├─ index.html                   ← MASTER + deployable page (edit this; it IS the site)
├─ vercel.json                  ← static hosting config
├─ build.sh                     ← re-syncs skill template + CSVs from index.html
├─ transfer.md                  ← full hand-off/spec (design rules, data state, how to continue)
├─ README.md                    ← this file
├─ initiatives_nodes.csv        ← reference: code,initiative,horizon (generated)
├─ initiatives_deps.csv         ← reference: source,target (generated)
└─ skill/                       ← installable Claude skill
   ├─ SKILL.md
   └─ assets/radial_template.html   (= index.html, generated by build.sh)
```

## Deploy to GitHub Pages
1. Push this folder's contents to a repo (keep `index.html` at the repo root).
2. Settings → Pages → Source: Deploy from a branch → `main` / `/root` → Save.
3. Open `https://<user>.github.io/<repo>/`.

Push commands (repo already created on GitHub):
```bash
cd radial-network
git init && git add . && git commit -m "Add radial network diagram"
git branch -M main
git remote add origin https://github.com/<user>/<repo>.git
git push -u origin main
```

## Deploy to Vercel
- CLI: `vercel` then `vercel --prod` from this folder.
- Or import the repo (preset "Other", static). Every push redeploys. `vercel.json` included.

## Editing
1. Edit `index.html` — data arrays `NODES` / `HZ` / `DEPS` and constants are near the top of the
   `<script>`.
2. Run `bash build.sh` to re-sync `skill/assets/radial_template.html` and both CSVs.
3. Re-deploy (git push).

In-page "Edit data ▸" edits are session-only (reset on refresh). To persist, edit the arrays in
`index.html` and re-deploy.
