Hand-off doc so this work can be resumed in Claude Code. Summarizes what was built, the design
rules, current data state, file inventory, and how to continue.

## What this is

An interactive **radial network diagram** of a corporate strategy "house" — 39 strategic
initiatives (SK Shieldus Physical Security Value-up, BCG-partnered) laid out on concentric rings
by time horizon, with cross-initiative dependency links. Single self-contained HTML file (D3 via
CDN), plus an installable Claude skill that regenerates it from data, plus a deploy folder for
GitHub Pages / Vercel.

Source material: two BCG slides — a "house" framework (Vision → Value Creation Model → Operating
Model, 39 initiatives coded 1a…6i, shaded by horizon) and a "Cross-initiative dependencies" page.

## Data model (three arrays near top of the `<script>`)

1. **NODES** — `["code","initiative label"]`. Leading char of code = category (1–6), used ONLY for
   angular ordering (groups same-family nodes into a contiguous arc). Not drawn/colored.
2. **HZ** — `{ "code": "short" | "mid" | "long" }`. Horizon drives BOTH color AND which ring.
   Applied via `NODES.forEach(n=>n.h=HZ[n.id])`.
3. **DEPS** — `["source","target"]` directed dependency pairs. Links drawn between initiatives only.

## Design rules (converged through visual debugging — do not regress)

**Layout**
- Three concentric rings by horizon, nearer = smaller: `HZR = {short:260, mid:440, long:620}`.
  Node radius = `HZR[n.h]`.
- Angle preserved across all nodes: order by category then code, `angle = -π/2 + 2π·(i/N)`.
  Changing a node's horizon changes only its radius, not its angle.
- No center hub, no grey spokes, no drawn category anchors. One faint guide circle per ring.
- Auto-fit `viewBox` to `root.getBBox()` + ~24px pad via `requestAnimationFrame`; re-`draw()` on resize.

**Color — by horizon (NOT category)**
- short `#22A3DF` (light blue), mid `#C3CDD4` (light grey; use dark code text `#1a2b38`),
  long `#5B6B76` (dark grey). Legend = Short/Mid/Long.

**Nodes**
- `NODE_R = 24`, fill `HZCOL[n.h]`, white stroke. Code text centered `dy:.32em`, bold ~16px.

**Labels (the hard part)**
- Two lines, balanced by word count (split minimizing length diff); 1 word → 1 line.
- Rotated to radial angle: `rotate(deg)`, deg = angle·180/π. Left half (`deg>90||deg<-90`):
  add `rotate(180)` + `text-anchor:end`; else `start`.
- **Always offset OUTWARD**: `rotate(deg) translate(OFF,0) [rotate(180) if flip]` — a SINGLE
  positive `translate(OFF,0)` for both halves. (Bug history: using `-OFF` on the left half sent
  text back across its own node. Never reintroduce the sign flip.)
- `OFF = NODE_R + GAP`, `GAP = 14`. Block vertically centered on radial line:
  `y0 = -(lines-1)*lh/2`, `lh = 20`, label font 19px.

**Links**
- Quadratic bowing toward center: `M sx,sy Q (sx+tx)*0.3,(sy+ty)*0.3 tx,ty`.
- Width 2.6px base, 4px hover; color `#7fb8dd`, opacity ~.55. Only if both endpoints exist.

**Tooltip**
- Hover/click dims non-adjacent nodes, highlights incident links.
- **Single merged line** `Dependency: <codes>` (incoming + outgoing, deduped) — NOT separate
  enables/depends lines. Font ~17px.

**Data table (always output)**
- Always-visible table below the diagram: Code (color badge) · Initiative · Horizon · Dependency
  (merged). Re-renders each `draw()` via `renderDataTable()`; Hide/Show toggle.
- Editable side panel ("Edit data ▸"): contenteditable Code/Initiative/Horizon; edits redraw and
  re-ring instantly. `source,target` textarea + Apply rebuilds DEPS. Edits are session-only.
- Export CSV: `initiatives_nodes.csv (code,initiative,horizon)` + `initiatives_deps.csv (source,target)`.

**Interaction**: `d3.zoom` scroll/drag ~[.4,4]; click node to isolate; click empty to clear.

## Deployment

Static site, no backend. The whole folder is the site (`index.html` at root):
- **GitHub Pages**: push folder contents to repo root → Settings → Pages → Source: `main` → open
  `https://<user>.github.io/<repo>/`.
- **Vercel**: import repo (preset "Other", static) or `vercel --prod` from the folder. `vercel.json`
  included (`cleanUrls`).

Git push (repo `dependency` already on GitHub):
```bash
cd radial-network
git init && git add . && git commit -m "Add radial network diagram"
git branch -M main
git remote add origin https://github.com/<user>/dependency.git
git push -u origin main
```
The data table travels inside index.html — nothing extra to configure.

## How to continue (typical next tasks)

- **Change a horizon**: edit `HZ` in `index.html`, run `bash build.sh` (re-syncs skill + CSVs).
- **Change dependencies**: edit `DEPS` in `index.html` (or edit in-browser → Export CSV → paste
  back), then `bash build.sh`. Remember the 5a / 5f / 6e-6i rules above.
- **Restyle**: all constants (HZR, NODE_R, fonts, GAP, colors) are near the top of the script and
  in the SKILL.md defaults table.
- **Verify visually** after any change with headless chromium screenshot before delivering:
  ```python
  from playwright.sync_api import sync_playwright; import pathlib
  url="file://"+str(pathlib.Path("index.html").resolve())
  with sync_playwright() as p:
      b=p.chromium.launch(); pg=b.new_page(viewport={"width":1400,"height":1000})
      pg.goto(url); pg.wait_for_timeout(1500); pg.screenshot(path="check.png"); b.close()
  ```
  (`pip install playwright --break-system-packages && playwright install chromium`)
  Check: no text–node overlap, three rings clearly separated, table renders 39 rows.
